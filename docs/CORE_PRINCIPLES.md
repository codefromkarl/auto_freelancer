# 项目核心准则（Core Principles）

本文件定义 BidMaster 的强制执行准则，优先级高于“流程顺畅”与“自动化便利”。

## 1) 失败优先（Fail-Closed）

**宁可禁止提交，也不要提交不合格数据。**

适用范围：
- 投标提案生成与提交
- 风控校验与质量门禁
- 外部 API 返回不完整时的本地落库

执行要求：
- 任何关键质量检查失败时，必须阻断提交（禁止“带病放行”）。
- 任何“中间态草稿/回退模板”不得作为最终提案直接提交到平台。
- 如果无法确认数据完整性或语义合规性，默认拒绝提交并提示人工处理。

## 2) 证据优先（Evidence Before Claim）

- 任何“成功/完成/已修复”声明，必须先有命令输出或可复现证据。
- 以实际执行结果为准，不以主观判断替代验证。

## 3) 一致性优先（Rule Consistency）

- 生成规则、校验规则、提交流程三者必须一致。
- 不允许“生成可通过，提交却被拦截”的长期不一致状态。
- 如存在差异，必须修复链路，而非放宽门禁。
