# AI 项目评分体系 (0-10分)

## 评分维度

| 维度 | 权重 | 分值范围 | 说明 |
|--------|--------|-----------|------|
| **预算合理性** | 25% | 0-2.5 | 预算是否合理，与工作量匹配 |
| **竞争程度** | 20% | 0-2.0 | 竞标人数，适中的竞争最好 |
| **描述清晰度** | 20% | 0-2.0 | 需求是否清晰明确 |
| **客户活跃度** | 15% | 0-1.5 | 客户是否在线活跃 |
| **技术匹配度** | 15% | 0-1.5 | 是否匹配你的技术栈 |
| **项目风险** | 5% | 0-0.5 | 是否有付款历史、是否是新手客户 |
| **总分** | 100% | 0-10 | 所有维度加权总和 |

---

## 详细评分规则

### 1. 预算合理性 (0-2.5分)

| 预算类型 | 评分标准 |
|----------|----------|
| **固定预算项目** | |
| - 预算过低 (<$10) | 0.0分 |
| - 预算偏低 ($10-$50) | 0.5分 |
| - 预算一般 ($50-$200) | 1.0分 |
| - 预算合理 ($200-$1000) | 2.0分 |
| - 预算优秀 (>$1000) | 2.5分 |
| **时薪项目** | |
| - 时薪过低 (<$10/h) | 0.0分 |
| - 时薪一般 ($10-$20/h) | 1.0分 |
| - 时薪良好 (>$20/h) | 2.0分 |

### 2. 竞争程度 (0-2.0分)

| 竞标人数 | 评分 | 说明 |
|----------|------|------|
| 0-5人 | 0.5分 | 竞争少，但可能是小项目 |
| 6-15人 | 2.0分 | 理想竞争范围 |
| 16-30人 | 1.5分 | 竞争激烈 |
| 31-50人 | 1.0分 | 非常激烈 |
| >50人 | 0.5分 | 过度竞争，不值得 |

### 3. 描述清晰度 (0-2.0分)

| 指标 | 评分 | 说明 |
|------|------|------|
| 描述长度 > 200字符 | +0.5分 | 有一定细节 |
| 描述长度 > 500字符 | +1.0分 | 描述详细 |
| 包含具体需求关键词 | +0.5分 | 明确要求 |
| 包含附件/文件 | +0.5分 | 有参考材料 |

### 4. 客户活跃度 (0-1.5分)

| 指标 | 评分 |
|------|------|
| 客户在线/最近活跃 | +1.0分 |
| 有完成项目记录 | +0.5分 |
| 评价良好 (>4星) | +0.5分 |
| 无历史记录 | +0.0分 |

### 5. 技术匹配度 (0-1.5分)

根据你的技术栈评估（需要配置）：
| 匹配度 | 评分 |
|--------|------|
| 完全匹配 | +1.5分 |
| 部分匹配 | +1.0分 |
| 可以学习 | +0.5分 |
| 不匹配 | +0.0分 |

### 6. 项目风险 (0-0.5分)

| 风险因素 | 扣分 |
|----------|--------|
| 新手客户（<5个项目） | -0.2分 |
| 未验证身份 | -0.2分 |
| 付款历史差 | -0.3分 |
| 正常客户 | +0.0分 |

---

## 总分等级定义

| 分数范围 | 等级 | 建议 |
|----------|------|------|
| 8.0-10.0 | S | 强烈推荐投标，立即行动 |
| 6.0-7.9 | A | 推荐投标，优先处理 |
| 4.0-5.9 | B | 可以考虑，需要权衡 |
| 2.0-3.9 | C | 不太推荐，除非特殊情况 |
| 0.0-1.9 | D | 不推荐，建议跳过 |
